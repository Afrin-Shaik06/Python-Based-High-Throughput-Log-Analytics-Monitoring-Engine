# High Throughput Log Analytics & Monitoring System

## 📌 Overview

This project models a **distributed log analytics and monitoring system** designed to process **high-throughput log streams** generated by applications and servers.

The system demonstrates **system architecture design, distributed computing, anomaly detection, and real-time monitoring** using modern data processing tools.

The project is divided into two milestones.

### Milestone-1

* Architecture design
* Log schema definition
* Distributed computing simulation

### Milestone-2

* Real-time log pipeline
* Distributed processing
* Anomaly detection
* Monitoring dashboards

---

# 🧩 System Components

• **Log Sources** – Applications or servers generating system logs

• **Ingestion Layer** – Collects logs and sends them to the processing pipeline

• **Distributed Processing Layer** – Processes logs using distributed computing frameworks

• **Processing Engine** – Parses raw logs into structured format

• **Anomaly Detection Engine** – Detects abnormal log patterns

• **Storage Layer** – Stores processed logs for querying

• **Monitoring Layer** – Visualizes logs through dashboards

Distributed computing is implemented using:

* Dask
* Ray

Monitoring dashboards are built using:

* Streamlit
* Kibana with Elasticsearch

# 🧾 Milestone-2 Components

### Distributed Cluster

`dask_cluster.py` initializes a distributed computing cluster to enable parallel processing of large log datasets.

### Log Generation

`generate_logs.py` simulates application logs used to test the pipeline.

### Log Ingestion

`ingestion.py` collects log data and feeds it into the processing system.

### Distributed Log Processing

`dask_pipeline.py` processes log data using the distributed cluster.

### Anomaly Detection

`anomaly_detection.py` analyzes processed logs to detect abnormal patterns.

### Alerting System

`alerting.py` generates alerts when anomalies are detected.

### Real-Time Simulation

`realtime_ingestion.py` simulates real-time log streaming.

### Elasticsearch Integration

`send_logs_to_elasticsearch.py` sends processed logs to Elasticsearch for indexing and analysis.

### Monitoring Dashboard

`dashboard.py` launches a Streamlit dashboard for monitoring log activity.
